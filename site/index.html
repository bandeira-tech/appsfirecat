How It Works

Three simple concepts power decentralized hosting

1

Build & Sign

Build your frontend with any tool. Sign it with your app's keypair. Your identity, your content.

2

Store in B3nd

Upload to the decentralized B3nd network. Content-addressed, immutable, and always available.

3

Serve via Hosts

Anyone can run a host server. Point it at your content. Instant global distribution.

Developer                     B3nd Network                    Host Servers
    |                              |                               |
    |  npm run build               |                               |
    |  sign & upload ------------->|                               |
    |                              |                               |

    |                              |                               |
Users --------------------------------------------------------> HTTP request
    |                              |                               |

For Service Providers

Run Hosting Infrastructure, Your Way

Become a hosting provider without building the entire stack.

The Provider Model

Run host servers that serve content from B3nd. Accept hosting requests from developers. You control what you serve.

1 Developer requests hosting via your inbox
|
2 You approve by creating a link
|
3 Your host serves their content

Why Become a Provider?

  • Minimal infrastructure - just run the host server
  • Content stored in B3nd - not your problem
  • Multi-tenant by default - host thousands of sites
  • Set your own terms and pricing
  • Geographic distribution - run hosts anywhere
  • Composable - combine with CDNs, edge networks

Start a Host Server

# Configure your target
export TARGET="mutable://accounts/YOUR_PUBKEY/hosted/"
export BACKEND_URL="https://testnet-evergreen.fire.cat"

# Run the host
deno run -A packages/public-static/mod.ts

# Now serving at http://localhost:8080
# Requests to /user1/* follow links to user content

Under the Hood

Simple primitives, powerful composition

URI-Based Routing

Content lives at B3nd URIs. Hosts map HTTP paths to URIs. Simple, predictable, debuggable.

immutable://accounts/{pubkey}/site/index.html

Link Following

A URI string stored at a path becomes a link. Hosts follow links automatically. Enables delegation and multi-tenancy.

mutable://provider/hosted/user1 -> immutable://user1/site/

Mutable Pointers

Update what's live by updating a pointer. Old builds stay available. Rollback = update pointer.

mutable://accounts/{pubkey}/target -> current build URI

Signed Content

All writes to accounts are signed. Verify who published what. Cryptographic proof of origin.

{ auth: [{ pubkey, signature }], payload: content }
